Órdenes de Servicio

← Volver a inicio

Objetivo

Registrar y controlar las órdenes de servicio, a través de un flujo estandarizado de estados y prioridades, para dar seguimiento completo a las reparaciones realizadas en el taller.

Usuarios del subsistema

  • Cliente
  • Personal técnico
  • Administrador

Requerimientos por usuario

Requerimientos del Cliente

  • Consultar el estado de su orden de servicio.
    Campos: ID Orden, ID Cliente.
  • Recibir notificaciones de cambios en el estatus.
    Campos: ID Orden, Estatus, Fecha Estimada de Entrega.
  • Visualizar fecha estimada de entrega.
    Campos: ID Orden, Fecha Estimada de Entrega, Estatus.

Requerimientos del Personal Técnico

  • Registrar diagnósticos de equipos.
    Campos: ID Orden, ID Equipo, Estado Físico Inicial, Notas.
  • Actualizar estatus de la orden.
    Campos: ID Orden, Estatus, Notas, Prioridad.
  • Registrar accesorios recibidos y estado físico inicial.
    Campos: ID Orden, Accesorios Recibidos, Estado Físico Inicial, Fecha de Recepción.

Requerimientos del Administrador

  • Registrar nuevas órdenes de servicio.
    Campos: ID Orden, ID Cliente, ID Equipo, Fecha de Recepción, Accesorios Recibidos, Estado Físico Inicial, Prioridad, Estatus, Fecha Estimada de Entrega, Notas.
  • Leer información de órdenes activas e históricas.
    Campos: ID Orden, ID Cliente, ID Equipo, Estatus, Prioridad, Fechas de Recepción y Entrega, Notas.
  • Actualizar datos de la orden.
    Campos: ID Orden + cualquier otro campo editable (Cliente, Equipo, Accesorios, Estado Físico, Prioridad, Estatus, Fecha de Entrega, Notas).
  • Borrar órdenes de servicio (*solo administrador).
    Campos: ID Orden.
